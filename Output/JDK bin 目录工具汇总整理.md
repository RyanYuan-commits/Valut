# 🔖 详细解释

## 1	开发与打包相关

### 1.1	javac 源代码编译工具

> Java 源代码编译器, 将 .java 文件编译为 .class 字节码文件, 是真正把源码变成可由 JVM 执行的第一步. 常用用法如 javac Hello. java, 支持指定类路径, 编码, 目标版本等参数.

```sh
$ javac <options> <java_file>
```

| 选项       | 备注                                     |
| -------- | -------------------------------------- |
| -d       | 用于指定编译生成的 .class 文件的**存放位置**.          |
| -cp      | 用于指定查找用户类文件的路径, 如果代码依赖了第三方库就必须使用该选项指定. |
| -verbose | 输出详细的**编译信息**, 包括正在加载的类, 编译的源文件等.      |

### 1.2	javap 反汇编工具

> Java 字节码反汇编工具, 用于查看 . class 文件的结构和方法签名. 常用于学习或排查编译结果, 例如查看编译器生成的默认构造函数, 泛型擦除后的签名等.

```sh
$ javap <options> <classes>
```

| 选项         | 说明                                              |
| ---------- | ----------------------------------------------- |
| -c         | 对代码进行反汇编, 输出每个方法的操作码.                           |
| -verbose   | 输出详细信息, 包括行号表, 局部变量表, 常量池, 异常表, 反汇编代码等几乎所有信息.   |
| -l         | 输出行号表和局部变量表. 通常已包含在 -v 的输出中.                    |
| -p         | 显示包括 private 在内的所有类成员和方法, 默认情况下 javap 不会显示私有成员. |
| -s         | 输出内部类型签名, 用于泛型等信息.                              |
| -sysinfo   | 显示正在处理的类的系统信息访问权限, 版本等.                         |
| -constants | 显示 static final 常量.                             |
