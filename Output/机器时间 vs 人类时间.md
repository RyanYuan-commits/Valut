# 🌐 核心观点

Java 8 的时间模型基于一个核心假设: **时间有两种完全不同的表现形式.**

---

# 🔖 详细解释

**机器时间**是连续的, 线性的, 像数轴一样从一个**基准点** (Epoch) 向两端延伸, **人类时间**是离散的, 循环的, 带有**地理和文化属性**.

## 1	机器时间: Instant

Instant 有 "某一特定时刻" 的意思, 在 Java 中用于表示机器时间, 是一个纳秒级精度的数字.

始终以 **UTC** (协调世界时) 为准, **不随地理位置改变**, 是**绝对**的, 无论你在上海还是纽约, 此刻的 `Instant. now()` 在逻辑上指向的是同一个宇宙瞬间, 适用于数据库主键, 日志记录, 计算代码执行耗时, 后端内部逻辑传递等场景.

## 2	人类时间: LocalXXX

是对日历系统的描述, 也就是所谓的 "墙上时间", 会根据当前的语境而改变, 在 Java 中对应 `LocalDate`, `LocalTime`, `LocalDateTime` 类.

人类时间是**模糊**的, 如果没有时区(ZoneId), `2026-01-01 08:00` 只是一个描述, **不是一个确定的时间点**.

## 3	人类时间 <-> 机器时间

机器时间与人类时间之间唯一的转换公式是:

> Instant + 时区 (ZoneId/ZoneOffset) = 人类时间 (ZonedDateTime)

如果没有时区这个 "映射表", 这两者之间无法互转.
