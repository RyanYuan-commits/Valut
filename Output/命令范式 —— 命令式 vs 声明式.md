# 🌐 核心观点

在现代软件开发中, 对两种范式的选择决定了代码的可维护性, 可读性以及抽象程度.

---

# 🔖 详细解释

## 1	核心定义: 控制流 vs 结果描述

| 维度   | 命令式                        | 声明式                                   |
| ---- | -------------------------- | ------------------------------------- |
| 哲学核心 | 如何做                        | 做什么                                   |
| 关注点  | 详细的算法步骤, 内存状态, 控制流         | 逻辑的关系, 数据的定义, 预期的结果                   |
| 执行逻辑 | 就像给机器人下单一连串的具体指令           | 就像写一份合同, 约定好输入和输出                     |
| 案例   | for 循环, if-else 分支, DOM 操作 | SQL 语句, HTML/CSS, Vue template, 函数式编程 |

## 2	深度拆解

### 2.1	命令式写法

```js
const numbers = [1, 2, 3, 4, 5];
const evens = [];

for (let i = 0; i < numbers.length; i++) {
  if (numbers[i] % 2 === 0) {
    evens.push(numbers[i]);
  }
}
```

开发者必须亲自管理变量的初始化, 循环的终止条件以及数组的推进逻辑.

### 2.2	声明式写法

```java
const numbers = [1, 2, 3, 4, 5];
const evens = numbers.filter(n => n % 2 === 0);
```

开发者直接描述 "我想要数组中满足偶数条件的元素".

## 3	声明式编程的三个层次

声明式并不是一种单一的技术, 它存在于多个维度:

**领域特定语言**:

- SQL: 你只需说 SELECT name FROM users, 而不需要写如何打开数据文件和遍历 B 树.
	
- CSS: 你只需说 color: blue, 而不需要写如何操作显卡渲染像素.

**函数式编程**: 利用 map, reduce, filter 等高阶函数替代显式的 for 循环, 减少副作用(Side Effects).

**声明式 UI 框架**: Vue / React / SwiftUI: 通过“数据驱动视图”, 让开发者从繁琐的 DOM/组件生命周期操作中解脱.
