# 📰 阅读笔记

![[MySQL 客户端请求处理流程.png|700]]

- 连接管理: 客户端进程通过 TCP, 命名管道或共享内存, Unix 域套接字之一来与服务端进行连接, 每当有一个客户端进程连接到服务端时, MySQL 会为其分配一个线程. 
	
- 查询缓存: key 是完整的 select 语句, 对表的大部分操作都会导致缓存失效, 效率较低, 新版 MySQL 已删除此功能.
	
- 语法解析: 判断发送过来的 SQL 语句是否正确, 将 SQL 语句解析成 MySQL 能够认识的结构. 
	
- 查询优化: 对解析出来的结构进行优化, 确定最终的执行方案. 
	
- 存储引擎: 调用存储引擎完成最终的查询.
